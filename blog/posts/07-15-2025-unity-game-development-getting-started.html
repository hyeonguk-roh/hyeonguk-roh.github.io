Understanding Unity's Interface

Unity's interface consists of several key windows and panels that you'll use throughout development.

Key Windows

  • Scene View: Where you build and arrange your game world
  • Game View: Shows how your game will look to players
  • Hierarchy: Lists all objects in your current scene
  • Project: Shows all assets in your project
  • Inspector: Displays and allows editing of selected object properties

Your First C# Script

C# is the primary programming language used in Unity. Let's create a simple script to understand the basics.

Creating a Script

using UnityEngine;

public class PlayerController : MonoBehaviour
{
    [SerializeField] private float moveSpeed = 5f;
    
    void Update()
    {
        float horizontalInput = Input.GetAxis("Horizontal");
        float verticalInput = Input.GetAxis("Vertical");
        
        Vector3 movement = new Vector3(horizontalInput, 0f, verticalInput);
        transform.Translate(movement * moveSpeed * Time.deltaTime);
    }
}

Understanding the Code

  • MonoBehaviour: Base class for all Unity scripts
  • [SerializeField]: Makes private variables visible in the Inspector
  • Update(): Called every frame
  • Input.GetAxis(): Gets input from keyboard/gamepad
  • Time.deltaTime: Time since last frame (for smooth movement)

Game Objects and Components

In Unity, everything is a GameObject. GameObjects can have multiple components that define their behavior and appearance.

Common Components

  • Transform: Position, rotation, and scale
  • Mesh Renderer: Renders 3D meshes
  • Collider: Defines collision boundaries
  • Rigidbody: Adds physics simulation
  • Scripts: Custom C# components

Basic Game Mechanics

Let's implement some common game mechanics to get you started.

Player Movement

public class PlayerMovement : MonoBehaviour
{
    [SerializeField] private float speed = 5f;
    [SerializeField] private float jumpForce = 5f;
    
    private Rigidbody rb;
    private bool isGrounded;
    
    void Start()
    {
        rb = GetComponent<Rigidbody>();
    }
    
    void Update()
    {
        // Handle movement
        float horizontal = Input.GetAxis("Horizontal");
        float vertical = Input.GetAxis("Vertical");
        
        Vector3 movement = new Vector3(horizontal, 0f, vertical);
        transform.Translate(movement * speed * Time.deltaTime);
        
        // Handle jumping
        if (Input.GetKeyDown(KeyCode.Space) && isGrounded)
        {
            rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);
        }
    }
    
    void OnCollisionEnter(Collision collision)
    {
        if (collision.gameObject.CompareTag("Ground"))
        {
            isGrounded = true;
        }
    }
    
    void OnCollisionExit(Collision collision)
    {
        if (collision.gameObject.CompareTag("Ground"))
        {
            isGrounded = false;
        }
    }
}

Collecting Items

public class Collectible : MonoBehaviour
{
    [SerializeField] private int pointValue = 10;
    
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            // Add points to player
            GameManager.Instance.AddPoints(pointValue);
            
            // Play collection effect
            AudioManager.Instance.PlayCollectSound();
            
            // Destroy the collectible
            Destroy(gameObject);
        }
    }
}

Best Practices for Unity Development

Following best practices will make your Unity projects more maintainable and efficient.

Coding Standards

  • Use PascalCase for class names and public methods
  • Use camelCase for private variables and methods
  • Add comments to explain complex logic
  • Use [SerializeField] instead of public for inspector variables
  • Cache component references in Start() or Awake()

Performance Tips

  • Avoid using GameObject.Find() in Update()
  • Use object pooling for frequently created/destroyed objects
  • Minimize garbage collection by reusing objects
  • Use LOD (Level of Detail) for complex models
  • Optimize your textures and models
