CSS Units of Length: A Complete Guide

2025-07-14 Hyeonguk Roh

Understanding CSS units of length is fundamental to creating responsive, accessible, and maintainable web designs. From pixels to viewport units, each has its specific use case and behavior. This comprehensive guide will walk you through all the CSS length units and when to use them.

Absolute Units

Absolute units are fixed and don't change based on the user's device or preferences. They're useful for print stylesheets and when you need precise control.

Pixels (px)

The most commonly used absolute unit. One pixel equals one dot on the screen.

.element {
  width: 300px;
  height: 200px;
  font-size: 16px;
}

Use cases: Borders, small icons, when you need exact pixel-perfect control.

Points (pt)

Primarily used for print media. 1pt = 1/72 inch.

@media print {
  body {
    font-size: 12pt;
    line-height: 14pt;
  }
}

Relative Units

Relative units scale based on other properties or the viewport, making them ideal for responsive design.

Em (em)

Relative to the font-size of the element. If no font-size is set, it inherits from the parent.

.parent {
  font-size: 16px;
}

.child {
  font-size: 1.5em; /* 24px (16px × 1.5) */
  margin: 1em; /* 24px (uses the element's font-size) */
}

Use cases: Typography scaling, margins, and padding that should scale with text size.

Rem (rem)

Relative to the root element's font-size (usually the html element). More predictable than em.

html {
  font-size: 16px;
}

.element {
  font-size: 1.5rem; /* 24px (16px × 1.5) */
  margin: 2rem; /* 32px (16px × 2) */
}

Use cases: Typography, spacing, and layout that should scale consistently across the entire document.

Percentage (%)

Relative to the parent element's property value.

.container {
  width: 100%;
  max-width: 1200px;
}

.child {
  width: 50%; /* Half of parent's width */
  height: 100%; /* Full height of parent */
}

Viewport Units

Viewport units are relative to the viewport's dimensions, making them perfect for responsive design.

Viewport Width (vw)

1vw = 1% of the viewport width.

.hero {
  width: 100vw;
  height: 50vh;
  font-size: 5vw; /* Scales with viewport width */
}

Viewport Height (vh)

1vh = 1% of the viewport height.

.fullscreen {
  height: 100vh;
  width: 100vw;
}

Viewport Minimum (vmin) and Maximum (vmax)

vmin is the smaller of vw or vh, vmax is the larger.

.responsive-square {
  width: 50vmin; /* 50% of the smaller viewport dimension */
  height: 50vmin;
}

.large-text {
  font-size: 8vmax; /* 8% of the larger viewport dimension */
}

Modern CSS Units

Container Query Units (cqw, cqh, cqi, cqb)

New units that are relative to the container's dimensions, not the viewport.

.card {
  container-type: inline-size;
}

.card-title {
  font-size: 5cqw; /* 5% of container width */
}

Logical Units (logical-width, logical-height)

Units that adapt to writing mode and text direction.

.international {
  margin-block: 1rem; /* Vertical margin */
  margin-inline: 2rem; /* Horizontal margin */
  padding-block-start: 1rem;
  padding-inline-end: 2rem;
}

Best Practices

  • Use rem for typography: It provides consistent scaling and respects user preferences.
  • Use em for component spacing: When you want spacing to scale with the component's text size.
  • Use viewport units sparingly: They can cause issues on mobile devices with dynamic toolbars.
  • Combine units strategically: Use calc() to combine different units for optimal results.
  • Test across devices: Always test your responsive design on various screen sizes.

Common Patterns

/* Responsive typography */
html {
  font-size: clamp(14px, 1vw + 1rem, 18px);
}

/* Flexible containers */
.container {
  width: min(90%, 1200px);
  margin-inline: auto;
}

/* Responsive spacing */
.section {
  padding: clamp(2rem, 5vw, 4rem);
}

Conclusion

Choosing the right CSS unit depends on your specific use case. For modern web development, I recommend using rem for typography, em for component spacing, and viewport units for truly responsive layouts. Always consider accessibility and user preferences when making your choices.

Remember: the best unit is the one that makes your design work well across all devices and respects user preferences. Don't be afraid to mix and match units to achieve the perfect balance of control and flexibility.
