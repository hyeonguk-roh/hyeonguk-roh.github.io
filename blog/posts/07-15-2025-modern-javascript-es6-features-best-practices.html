Modern JavaScript: ES6+ Features and Best Practices

2025-07-15 Hyeonguk Roh

JavaScript has evolved significantly since its inception, with ES6 (ECMAScript 2015) marking a major milestone in the language's development. Modern JavaScript introduces powerful features that make code more readable, maintainable, and efficient. This comprehensive guide covers the essential ES6+ features and best practices for writing modern JavaScript.

What is ES6 and Why It Matters

ES6 (ECMAScript 2015) was a major update to JavaScript that introduced many new features and syntax improvements. Since then, JavaScript has continued to evolve with annual releases, bringing new capabilities and improvements to the language.

Key Benefits of Modern JavaScript

  • Improved Readability: Cleaner, more expressive syntax
  • Better Performance: Optimized language features
  • Enhanced Maintainability: Modular code organization
  • Reduced Bugs: Stricter syntax and better error handling
  • Modern Development: Support for contemporary programming patterns

Variable Declarations: let, const, and var

ES6 introduced `let` and `const` as alternatives to `var`, providing better scoping and preventing common programming errors.

const - For Constants

// const creates immutable bindings
const PI = 3.14159;
const API_URL = 'https://api.example.com';
const USER_ROLES = ['admin', 'user', 'guest'];

// const prevents reassignment
// PI = 3.14; // TypeError: Assignment to constant variable

// However, const doesn't make objects immutable
const user = { name: 'John', age: 30 };
user.age = 31; // This works - object properties can be modified
// user = {}; // This fails - can't reassign the variable

let - For Block-Scoped Variables

// let provides block scope
function example() {
    let x = 1;
    
    if (true) {
        let x = 2; // Different variable, block-scoped
        console.log(x); // 2
    }
    
    console.log(x); // 1
}

// Compare with var (function-scoped)
function varExample() {
    var x = 1;
    
    if (true) {
        var x = 2; // Same variable, function-scoped
        console.log(x); // 2
    }
    
    console.log(x); // 2
}

Arrow Functions

Arrow functions provide a concise syntax for writing function expressions and have lexical `this` binding.

Basic Arrow Function Syntax

// Traditional function
function add(a, b) {
    return a + b;
}

// Arrow function equivalent
const add = (a, b) => {
    return a + b;
};

// Implicit return for single expressions
const add = (a, b) => a + b;

// Single parameter doesn't need parentheses
const square = x => x * x;

// No parameters need empty parentheses
const getRandom = () => Math.random();

// Multiple lines need explicit return
const processData = (data) => {
    const filtered = data.filter(item => item.active);
    const sorted = filtered.sort((a, b) => a.name.localeCompare(b.name));
    return sorted;
};

Arrow Functions and this Context

// Traditional function - this context can change
function Timer() {
    this.seconds = 0;
    
    setInterval(function() {
        this.seconds++; // this is undefined or window
        console.log(this.seconds);
    }, 1000);
}

// Arrow function - preserves this context
function Timer() {
    this.seconds = 0;
    
    setInterval(() => {
        this.seconds++; // this refers to Timer instance
        console.log(this.seconds);
    }, 1000);
}

// Event handlers
const button = document.querySelector('button');

// Traditional function
button.addEventListener('click', function() {
    console.log(this); // button element
});

// Arrow function
button.addEventListener('click', () => {
    console.log(this); // window or undefined
});

Destructuring Assignment

Destructuring allows you to extract values from arrays and objects into distinct variables using a syntax that mirrors the construction of array and object literals.

Array Destructuring

// Basic array destructuring
const colors = ['red', 'green', 'blue'];
const [first, second, third] = colors;
console.log(first); // 'red'
console.log(second); // 'green'
console.log(third); // 'blue'

// Skipping elements
const [primary, , tertiary] = colors;
console.log(primary); // 'red'
console.log(tertiary); // 'blue'

// Rest operator
const [head, ...tail] = colors;
console.log(head); // 'red'
console.log(tail); // ['green', 'blue']

// Default values
const [a, b, c, d = 'yellow'] = colors;
console.log(d); // 'yellow'

// Swapping variables
let x = 1, y = 2;
[x, y] = [y, x];
console.log(x, y); // 2, 1

Object Destructuring

// Basic object destructuring
const user = {
    name: 'John Doe',
    age: 30,
    email: 'john@example.com',
    address: {
        city: 'New York',
        country: 'USA'
    }
};

const { name, age, email } = user;
console.log(name); // 'John Doe'
console.log(age); // 30

// Renaming variables
const { name: userName, age: userAge } = user;
console.log(userName); // 'John Doe'
console.log(userAge); // 30

// Nested destructuring
const { address: { city, country } } = user;
console.log(city); // 'New York'
console.log(country); // 'USA'

// Default values
const { name, age, role = 'user' } = user;
console.log(role); // 'user'

// Function parameters
function processUser({ name, age, email }) {
    console.log(`Processing ${name} (${age}) at ${email}`);
}

processUser(user); // Processing John Doe (30) at john@example.com

Template Literals

Template literals provide an easy way to create multiline strings and perform string interpolation.

// Basic template literals
const name = 'John';
const age = 30;
const greeting = `Hello, my name is ${name} and I am ${age} years old.`;

// Multiline strings
const html = `
<div class="user-card">
    <h2>${name}</h2>
    <p>Age: ${age}</p>
    <button onclick="editUser('${name}')">Edit</button>
</div>
`;

// Expressions in template literals
const items = ['apple', 'banana', 'orange'];
const list = `
<ul>
    ${items.map(item => `<li>${item}</li>`).join('')}
</ul>
`;

// Tagged template literals
function highlight(strings, ...values) {
    let result = '';
    strings.forEach((string, i) => {
        result += string;
        if (values[i]) {
            result += `<span class="highlight">${values[i]}</span>`;
        }
    });
    return result;
}

const highlighted = highlight`Hello ${name}, you are ${age} years old!`;

Spread and Rest Operators

The spread (`...`) operator allows you to expand elements, while the rest operator allows you to collect multiple elements into an array.

Spread Operator

// Array spreading
const numbers = [1, 2, 3];
const moreNumbers = [...numbers, 4, 5, 6];
console.log(moreNumbers); // [1, 2, 3, 4, 5, 6]

// Copying arrays
const original = [1, 2, 3];
const copy = [...original];
console.log(copy); // [1, 2, 3]
console.log(copy === original); // false

// Merging arrays
const fruits = ['apple', 'banana'];
const vegetables = ['carrot', 'lettuce'];
const food = [...fruits, ...vegetables];
console.log(food); // ['apple', 'banana', 'carrot', 'lettuce']

// Object spreading
const person = { name: 'John', age: 30 };
const employee = { ...person, id: 123, department: 'Engineering' };
console.log(employee); // { name: 'John', age: 30, id: 123, department: 'Engineering' }

// Copying objects
const originalObj = { a: 1, b: 2 };
const copyObj = { ...originalObj };
console.log(copyObj); // { a: 1, b: 2 }
console.log(copyObj === originalObj); // false

// Merging objects
const defaults = { theme: 'dark', language: 'en' };
const userPrefs = { theme: 'light' };
const settings = { ...defaults, ...userPrefs };
console.log(settings); // { theme: 'light', language: 'en' }

Rest Operator

// Function parameters
function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3, 4, 5)); // 15

// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];
console.log(first); // 1
console.log(second); // 2
console.log(rest); // [3, 4, 5]

// Object destructuring
const { name, age, ...otherProps } = {
    name: 'John',
    age: 30,
    city: 'New York',
    country: 'USA'
};
console.log(name); // 'John'
console.log(age); // 30
console.log(otherProps); // { city: 'New York', country: 'USA' }

Modules (ES6 Modules)

ES6 modules provide a standardized way to organize and share code between files.

Exporting from Modules

// math.js - Named exports
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
export const multiply = (a, b) => a * b;
export const divide = (a, b) => a / b;

// Default export
const PI = 3.14159;
export default PI;

// Alternative: export at the end
const utils = {
    formatCurrency: (amount) => `$${amount.toFixed(2)}`,
    formatDate: (date) => date.toLocaleDateString(),
    generateId: () => Math.random().toString(36).substr(2, 9)
};

export { utils as default };
export const { formatCurrency, formatDate, generateId } = utils;

Importing Modules

// Named imports
import { add, subtract, multiply } from './math.js';

// Default import
import PI from './math.js';

// Renaming imports
import { add as addNumbers, subtract as subtractNumbers } from './math.js';

// Import everything as namespace
import * as MathUtils from './math.js';
console.log(MathUtils.add(5, 3)); // 8

// Mixed imports
import PI, { add, subtract } from './math.js';

// Dynamic imports (ES2020)
async function loadModule() {
    const module = await import('./math.js');
    console.log(module.add(2, 3)); // 5
}

Async/Await

Async/await provides a more readable way to work with promises and asynchronous operations.

Basic Async/Await

// Traditional promise-based approach
function fetchUser(id) {
    return fetch(`/api/users/${id}`)
        .then(response => response.json())
        .then(user => {
            console.log(user);
            return user;
        })
        .catch(error => {
            console.error('Error fetching user:', error);
        });
}

// Async/await approach
async function fetchUser(id) {
    try {
        const response = await fetch(`/api/users/${id}`);
        const user = await response.json();
        console.log(user);
        return user;
    } catch (error) {
        console.error('Error fetching user:', error);
    }
}

// Multiple async operations
async function fetchUserAndPosts(userId) {
    try {
        const [user, posts] = await Promise.all([
            fetch(`/api/users/${userId}`).then(res => res.json()),
            fetch(`/api/users/${userId}/posts`).then(res => res.json())
        ]);
        
        return { user, posts };
    } catch (error) {
        console.error('Error fetching data:', error);
    }
}

Error Handling with Async/Await

// Proper error handling
async function processData() {
    try {
        const data = await fetchData();
        const processed = await processData(data);
        const result = await saveData(processed);
        return result;
    } catch (error) {
        if (error.name === 'NetworkError') {
            console.error('Network error occurred');
        } else if (error.name === 'ValidationError') {
            console.error('Data validation failed');
        } else {
            console.error('Unexpected error:', error);
        }
        throw error; // Re-throw to let caller handle it
    }
}

// Async functions in loops
async function processItems(items) {
    const results = [];
    
    // Sequential processing
    for (const item of items) {
        const result = await processItem(item);
        results.push(result);
    }
    
    // Parallel processing
    const promises = items.map(item => processItem(item));
    const parallelResults = await Promise.all(promises);
    
    return { sequential: results, parallel: parallelResults };
}

Classes

ES6 classes provide a more familiar syntax for creating objects and implementing inheritance.

// Basic class definition
class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    
    sayHello() {
        return `Hello, my name is ${this.name}`;
    }
    
    get info() {
        return `${this.name} is ${this.age} years old`;
    }
    
    set info(value) {
        [this.name, this.age] = value.split(' is ');
        this.age = parseInt(this.age);
    }
}

// Inheritance
class Employee extends Person {
    constructor(name, age, department) {
        super(name, age);
        this.department = department;
    }
    
    work() {
        return `${this.name} is working in ${this.department}`;
    }
    
    // Override method
    sayHello() {
        return `${super.sayHello()} and I work in ${this.department}`;
    }
}

// Static methods
class MathUtils {
    static add(a, b) {
        return a + b;
    }
    
    static multiply(a, b) {
        return a * b;
    }
}

console.log(MathUtils.add(5, 3)); // 8

Best Practices for Modern JavaScript

Following these best practices will help you write cleaner, more maintainable JavaScript code.

Code Organization

  • Use meaningful variable and function names
  • Keep functions small and focused on a single responsibility
  • Use modules to organize related functionality
  • Prefer composition over inheritance
  • Use consistent formatting and indentation

Performance Considerations

  • Avoid creating functions inside loops
  • Use appropriate data structures for your use case
  • Minimize DOM manipulations
  • Use debouncing for frequent events
  • Lazy load modules when possible

Error Handling

  • Always handle promise rejections
  • Use try-catch blocks for async operations
  • Provide meaningful error messages
  • Log errors appropriately for debugging
  • Implement graceful degradation

Conclusion

Modern JavaScript with ES6+ features has transformed the language into a powerful, expressive tool for web development. By mastering these features and following best practices, you can write cleaner, more maintainable, and more efficient code.

Remember that JavaScript continues to evolve, with new features being added regularly. Stay updated with the latest ECMAScript proposals and browser support to take advantage of the newest capabilities.

Start incorporating these features into your projects gradually, and you'll soon find yourself writing more elegant and powerful JavaScript code. Happy coding!
